# Codebase Feature Directory Plan

## The Pattern (same as `umamoe/`)

Move files that share the same feature into a dedicated directory.
Leave a one-liner **shim** at the original path so zero callers break:

```js
// Original path (e.g. tasks/milestones.js) becomes:
export * from '../milestone/milestones.js';
```

## Directories to Leave As-Is

| Directory | Reason |
|---|---|
| `commands/` | Command loader scans this folder by name — reorganizing risks the loader |
| `db/migrations.js` | Cross-cutting; used by every DB module at boot |
| `core/` (infra files) | `config.js`, `log.js`, `format.js`, `health.js`, `channels.js` etc. are infrastructure used everywhere — no clean feature cluster |
| `scripts/` | Dev/admin CLI tools; flat is fine |
| `handlers/` | Already has `handlers/features/` subfolder; event handlers stay here |
| `utils/imageReport*.js` | Shared rendering engine used by many features — not feature-specific |

---

## Recommended Order (lowest risk → highest)

1. `attendance/` — smallest, 2 files, fewest callers
2. `achievements/` — small, self-contained
3. `archive/` — no cross-feature deps
4. `onboarding/` — cohesive, few callers
5. `links/` — needs care (many commands import linksDb)
6. `leaderboard/` — moderate callers
7. `warnings/` — largest task cluster, needs careful import tracing
8. `milestone/` — most complex (milestone-tiers imported by many files)

Each step: read destination files → fix internal imports → write to new path → replace original with shim → verify bot starts clean.
