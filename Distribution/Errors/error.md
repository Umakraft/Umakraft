# Distribution Errors

## Purpose

This folder contains standardized distribution error definitions and messaging guidance.

## Responsibilities

- Define common error categories and types for Distribution.
- Standardize error handling across Interaction, Delivery, and Adapters.
- Provide clear, safe user-facing messages for failure cases.
- Surface failure reasons without exposing internal details.

## Notes

- Keep error definitions separate from business logic and data retrieval.
- Use error contracts consistently across Distribution components.
- Prefer predictable, actionable error responses for users and integrations.
