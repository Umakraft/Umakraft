# Image Report Blueprint

## Purpose

This blueprint defines the image-report product for the Workshop.

It describes the layout, components, visual structure, and presentation rules required to render a report image from pipeline data.

## Product overview

The image report is a snapshot card that summarizes a pipeline result and highlights the most important metrics, trends, and flags.

It is designed for automated monitoring, review dashboards, alerts, and archive snapshots.

## Layout

1. Header
   - Product title
   - Source / pipeline identifier
   - Timestamp
   - Status badge
2. Summary metrics band
   - Daily gain
   - Weekly gain
   - Monthly gain
   - Rank change
3. Main visual section
   - Trend sparkline or bar chart
   - Primary status icon
4. Flag and detail panel
   - Active flags
   - Key profile attributes
   - Notes or alert summary
5. Footer
   - Blueprint version
   - Generated by
   - Raw payload reference

## Component definitions

### Header

- `title`: string
- `subtitle`: string
- `generatedAt`: ISO timestamp
- `status`: enum(`OK`, `Warning`, `Alert`, `Unknown`)
- `sourceId`: string

### Summary metrics band

- `dailyGain`: numeric
- `weeklyGain`: numeric
- `monthlyGain`: numeric
- `rankChange`: numeric

### Main visual section

- `trendData`: array of { `label`: string, `value`: number }
- `primaryChartType`: enum(`sparkline`, `bar`, `line`)
- `icon`: enum(`up`, `down`, `flat`, `warning`)

### Flag and detail panel

- `flags`: array of {
  - `code`: string
  - `label`: string
  - `severity`: enum(`low`, `medium`, `high`)
  - `description`: string
  }
- `profileDetails`: array of { `label`: string, `value`: string }
- `note`: string

### Footer

- `blueprintVersion`: string
- `rendererVersion`: string
- `payloadReference`: string

## Canvas and styling

- Canvas dimensions: 1200x800 px (desktop card) or scalable SVG.
- Base font: 16px sans-serif.
- Palette: neutral background, high-contrast metric colors, status accent colors.
- Layout grid: 12-column responsive grid for card sections.
- Spacing: 24px outer margin, 16px internal gutters.

## Rendering rules

- Use an SVG primary output to preserve crisp text and vector shapes.
- Support export to PNG at 2x scale.
- Render text labels clearly with high contrast.
- Avoid more than 5 colors besides the base palette.
- Highlight active flags in the detail panel.
- Place the status badge in the top-right of the header.

## Data contract

The blueprint expects a JSON payload with these sections:

- `meta`
  - `id`
  - `generatedAt`
  - `source`
  - `status`
- `metrics`
  - `dailyGain`
  - `weeklyGain`
  - `monthlyGain`
  - `rankChange`
- `trend`
  - `points`: array of { `label`, `value` }
- `flags`
  - array of flag objects
- `profile`
  - array of detail key/value pairs

## Validation

A blueprint implementation must validate that:

- required fields are present
- numeric metrics are within reasonable bounds
- trend points are not empty
- flag entries include `code`, `label`, and `severity`

## Example payload

```json
{
  "meta": {
    "id": "report-001",
    "generatedAt": "2026-07-18T00:00:00.000Z",
    "source": "pipeline/uma-moe",
    "status": "Warning"
  },
  "metrics": {
    "dailyGain": 18,
    "weeklyGain": 72,
    "monthlyGain": 210,
    "rankChange": -2
  },
  "trend": {
    "points": [
      { "label": "Mon", "value": 82 },
      { "label": "Tue", "value": 90 },
      { "label": "Wed", "value": 105 },
      { "label": "Thu", "value": 98 },
      { "label": "Fri", "value": 110 }
    ]
  },
  "flags": [
    { "code": "HIGH_GAIN", "label": "High Gain", "severity": "medium", "description": "Daily gain exceeded the configured threshold." },
    { "code": "RANK_DROP", "label": "Rank Drop", "severity": "high", "description": "Rank decreased by 2 positions." }
  ],
  "profile": [
    { "label": "Name", "value": "Example Profile" },
    { "label": "Tier", "value": "Gold" }
  ]
}
```

## Notes

- This blueprint is intentionally format-focused. The Fabricator or Renderer is responsible for actual SVG/PNG generation.
- Keep the source data payload separate from the visual layout specification.
