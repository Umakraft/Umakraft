# Memberlist Blueprint

## Purpose

This blueprint defines the visual and data structure for the `/memberlist` deliverable.

It describes how a circle's member roster should be assembled, presented, and delivered after the command requests it.

## Product overview

The `/memberlist` report displays the full roster of a circle, showing each member's trainer identity, fan count, and rank. It is intended for at-a-glance circle management and recruitment review.

## Layout

1. Header
   - Report title: `Circle Member List`
   - Circle name / ID
   - Timestamp
   - Total member count
2. Member table
   - Rank position within circle
   - Trainer name / ID
   - Fan count
   - Fan gain (daily, if available)
3. Summary band
   - Total circle fans
   - Average fans per member
   - Highest-ranked member callout
4. Footer
   - Blueprint version
   - Source metadata
   - Generated by UmaKraft command

## Component definitions

### Header

- `title`: string
- `circleName`: string
- `circleId`: string
- `generatedAt`: ISO timestamp
- `memberCount`: number

### Member table rows

- `members`: array of {
  - `rank`: number
  - `trainerId`: string
  - `trainerName`: string
  - `fans`: number
  - `dailyGain`: number or null
  }

### Summary band

- `totalFans`: number
- `averageFans`: number
- `topMember`: string (trainer name or id)

### Footer

- `blueprintVersion`: string
- `source`: string
- `command`: `/memberlist`

## Data contract

The blueprint expects a payload with these sections:

- `meta`
  - `circleId`
  - `circleName`
  - `generatedAt`
  - `memberCount`
- `roster`
  - `members`: array of member row objects
- `summary`
  - `totalFans`
  - `averageFans`
  - `topMember`

## Workflow

```text
Discord User
      │
      ▼
  /memberlist
      │
      ▼
Command validates request
      │
      ▼
Distribution/Retriever fetches approved memberlist product
      │
      ▼
Delivery renders the report using the Memberlist blueprint
      │
      ▼
Discord Response
```

## Validation

A blueprint implementation must ensure:

- `members` array is present and non-empty
- each member row includes `trainerId` and `fans`
- summary totals are consistent with member rows
- the command metadata matches `/memberlist`

## Notes

- The member table should be ordered by fan count descending (rank ascending) by default.
- Discord embed field limits apply — paginate or truncate if member count exceeds 25.
- The blueprint specifies structure; rendering is handled by Delivery.
