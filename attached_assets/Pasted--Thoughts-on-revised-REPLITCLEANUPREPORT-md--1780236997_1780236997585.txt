⚠️ **Three things that are still off**

1. uma.js "Add caching layer" recommendation is wrong.
   uma.js already has a snapshot cache — it's the core
   of how the bot avoids hammering uma.moe every command.
   This recommendation would mislead a future agent into
   thinking something is broken that isn't. Remove it or
   replace with "Cache exists — verify TTL is appropriate."

2. Runtime Activity Report presents guesses as facts.
   "Recently Active Systems" and "Frequently Used Commands"
   list specific files and commands as if confirmed by logs.
   They're not — they're reasonable guesses. A future agent
   reading this could treat /fan_gain as confirmed high-usage
   and deprioritize /admin_setjoindate — based on nothing.
   Fix: label the whole section clearly as ESTIMATED / UNVERIFIED
   or just remove the example lists entirely.

3. UNREGISTERED COMMANDS is vague where it could be specific.
   The section says "verify against core/deploy-commands.js"
   but I already identified the gap:
   replit.md documents 13 commands.
   commands/ folder has 23 files.
   The 10 undocumented ones are:
     admin_setjoindate, admin_syncCards, intercircleleaderboard,
     joindate, keep, status, test_milestone, total_circlefan_gain,
     total_fan, admin_sync
   These should be listed explicitly — vague instructions get
   ignored, specific lists get acted on.
