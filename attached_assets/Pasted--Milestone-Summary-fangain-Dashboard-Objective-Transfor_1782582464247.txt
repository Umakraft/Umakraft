# Milestone Summary — `/fangain` Dashboard

## Objective

Transform `/fangain` into the flagship command of the bot by providing a professional image-based dashboard that instantly answers:

> **"How am I performing today?"**

The dashboard should prioritize actionable information while remaining clean, mobile-friendly, and fully data-driven.

---

# Current Architecture

The existing bot already provides a strong foundation:

* Modular command system
* Repository pattern
* Database abstraction
* Scheduler / dataSync
* Image report rendering
* Chromium HTML rendering
* Leaderboards
* Fan gain tracking
* Discord slash commands

This milestone focuses on extending the current architecture rather than rebuilding it.

---

# Phase 0 — Data Foundation

Before expanding analytics, verify the reliability of the underlying data.

Tasks:

* Verify dataSync correctly stores daily snapshots.
* Validate Daily / Weekly / Monthly fan gain calculations.
* Ensure Fan Gain Protection correctly initializes new links.
* Verify scheduler reliability after bot restarts.
* Optimize database indexes for leaderboard performance.
* Define snapshot retention policy.

Goal:

A reliable historical dataset that future analytics can safely depend on.

# Technical Constraints

Confirmed limitations:

## Hourly Progress Chart

Not feasible.

Reason:

uma.moe does not expose hourly fan gain history.

Instead, use stored daily snapshots for weekly and monthly visualizations.

---

## Fan Gain Streak

Cannot reuse the existing Discord login streak.

Requires a separate completion tracking system.

---

## Completion Rate

Requires historical daily completion records.

---

## Personal Best

Requires new database columns and update logic.

---

# Design Principles

* Every bot response is an Image Report.
* Never display statistics that cannot be verified.
* Prioritize actionable information over excessive detail.
* Keep the default dashboard concise.
* Reserve advanced analytics for autocomplete reports.
* Build all future features on reliable historical snapshot data.
* Ensure all analytics remain accurate, consistent, and maintainable.

---

# Milestone Goal

Deliver a polished `/fangain` dashboard that becomes the primary command members use every day.

The report should immediately communicate:

* Current progress
* Target completion
* Daily, Weekly, and Monthly fan gain
* Rankings
* Competition
* Performance trends
* Guild progress

Future milestones will expand this foundation into a complete analytics platform featuring achievements, historical statistics, advanced charts, and automated guild reporting.
